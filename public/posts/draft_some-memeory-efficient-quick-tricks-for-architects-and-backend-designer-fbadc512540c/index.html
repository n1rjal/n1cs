Some memeory efficient quick tricks for architects and backend designer

Lexically sortable Values

Date and time can be lexcially sorted. The following conditions are true. For ISO string as well. That means you don’t have to parse number from ISO date to compare for greater time based values. Learn more about lexical sorting here.

Hashing over Ecluidian Distance

Whenever you are trying to build a geo proximity like feature. Always use tricks like geo hash and quad tree. That makes it easier to build a geo proximity service and is more efficient than using distance algorithm everytime you query the database. You can utilize btree indexes and the performance will be better and no dynamic calculation is needed.
